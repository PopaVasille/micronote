# Micronote – Project Guidelines

Short, practical notes to get productive quickly.

- Tech stack
  - Backend: PHP 8.2, Laravel 12, Sanctum, Inertia (server), Ziggy
  - Frontend: Vue 3 + Inertia (client), Vite 6, Tailwind CSS, Axios, vue-i18n
  - Tooling: Composer, Node, PHPUnit 11, Laravel Pint, Pail (logs), queue worker

- Project structure (key paths)
  - app/ … Laravel application code (Controllers, Models, Policies, Jobs, etc.)
  - routes/web.php … HTTP routes (Inertia pages) | routes/auth.php … auth
  - resources/views/app.blade.php … Inertia root + @vite
  - resources/js/app.js … Vite entry; Inertia app bootstrapping
  - resources/js/Pages/*.vue … Vue pages resolved by Inertia
  - resources/js/i18n.js + resources/js/locales/*.json … i18n messages
  - tests/Feature, tests/Unit … PHPUnit tests
  - vite.config.js, tailwind.config.js, postcss.config.js … build tooling

- Local setup (Windows/PowerShell examples)
  1) Requirements: PHP 8.2+, Composer, Node 18+.
  2) Install deps:
     - composer install
     - npm install
  3) Environment:
     - Copy .env.example to .env and adjust DB/APP_URL/etc.
     - php artisan key:generate
  4) Database: php artisan migrate --seed (optional seeders if present)

- Run the app
  - Everything at once (PHP server, queue, logs, Vite): composer run dev
  - Or individually:
    - php artisan serve
    - php artisan queue:listen --tries=1
    - php artisan pail --timeout=0  (developer-friendly log viewer)
    - npm run dev  (Vite dev server)
  - Production build: npm run build (outputs to public/build via Vite plugin)

- Routes and pages
  - Define routes in routes/web.php using Inertia::render('PageName') or controller actions that return Inertia.
  - Place Vue pages in resources/js/Pages/PageName.vue; they are resolved dynamically from app.js.
  - Do not edit public/build directly; it’s generated by Vite.

- i18n guidelines
  - Base messages: resources/js/locales/en.json, ro.json
  - Sectioned files: privacy.en.json, terms.ro.json, etc. are auto-merged in i18n.js
  - When adding new sections, follow the existing glob patterns in i18n.js

- Testing
  - Run all tests: composer test  (clears config then runs php artisan test)
  - PHPUnit uses in-memory SQLite (see phpunit.xml). No local DB needed for tests.
  - Examples:
    - php artisan test --filter=ExampleTest
    - php artisan test tests/Feature/ExampleTest.php

- Code quality
  - PHP style: vendor\bin\pint --test (check) | vendor\bin\pint (fix)
  - Prefer typed code and SOLID patterns; keep controllers thin, use Services/Actions where helpful.
  - Frontend: keep components small; co-locate composables/utilities by feature when it aids clarity.

- Common artisan commands
  - Make files: php artisan make:controller NameController, make:model Name -mcr, make:job, make:policy
  - DB: php artisan migrate, migrate:rollback, tinker for quick experiments

- CI/local checks before commit (suggested)
  - composer test
  - vendor\bin\pint --test
  - npm run build (when touching frontend build or assets)
